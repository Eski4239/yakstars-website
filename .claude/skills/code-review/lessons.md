# Code review lessons learned

Living log of recurring mistakes found in this repo, so the code-review skill stops repeating them. Read this file at the start of every review. Append a new entry only for a genuinely recurring pattern (not a one-off typo) after fixing it.

Entry format:

```
## <short title>
- **Pattern:** what went wrong (generic enough to recur)
- **Severity:** Critical/High/Medium/Low
- **Fix:** what the correct fix looks like
- **First seen:** <date>, file/context
```
